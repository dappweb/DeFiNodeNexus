测试目标：覆盖用户在网页端可触达的全部核心业务逻辑，输出“业务对应关系 + 测试步骤 + 预期结果（页面表现与链上状态）”。

覆盖页面： Home Nodes Swap Earnings Team Admin

1. 测试前置条件

  • 钱包角色：Owner、用户A、用户B、用户C、交易员（至少5个地址）。
  • 合约地址已配置：NEXUS、SWAP、TOT、TOF、USDT（前端环境变量可读）。
  • 用户持有测试资产：USDT/TOF/TOT，且有足够 CNC 支付 Gas。
  • 至少配置 1 个 NFTA 档位、3 个 NFTB 档位；Swap 已注入流动性。
  • 若要验证公告接口：/api/announcements 可返回列表，且 Admin 有发布权限。

2. 业务—页面覆盖映射

4.1 功能通过

  • 上述 52个用例 全部执行，关键路径通过率 100%。
