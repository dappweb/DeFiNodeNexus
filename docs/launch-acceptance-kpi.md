# 上线验收 KPI（完成版）

更新时间：2026-03-26  
适用对象：产品 / 测试 / 前端 / 合约 / 运营

## 1. 验收目标

本文件用于上线前与上线后 7 天的 Go / No-Go 统一判定，聚焦三类结果：
- 关键路径是否跑通（可用性）
- 用户是否顺利完成核心行为（转化）
- 交易是否稳定且可恢复（可靠性）

## 3. 关键指标计算公式

1) 交易成功率  
`交易成功率 = 成功上链交易数 / 发起交易总数 × 100%`

2) 首次关键路径完成率  
`首次关键路径完成率 = 24h 内完成连接+绑定+首次买入的新钱包数 / 当日新连接钱包数 × 100%`

3) 可预防失败占比  
`可预防失败占比 = (余额不足 + 授权不足 + 参数非法)失败数 / 总失败数 × 100%`

4) 授权补救成功率  
`授权补救成功率 = allowance 不足后完成 approve 并在 10 分钟内交易成功的用户数 / allowance 不足用户数 × 100%`

---

## 4. 验收执行步骤（发布当日）

1. 执行自动化：`npm run test:e2e`，记录通过率（P0-1）  
2. 执行真实链路：Round3（可重试 5 次），记录 `EXIT`（P0-2）  
3. 使用 3 个账号走完整主流程并留证据（P0-3）  
4. 汇总失败类型，计算可预防失败占比（P0-4）  
5. 跨端（桌面 + 移动）验证页面可交互（P0-5）

> 判定规则：P0-1 ~ P0-5 任一不达标即 No-Go。

## 6. 当前项目落地建议（最小改动）

- 无埋点情况下先完成 P0 全量验收，保证可发布。  
- 发布后优先补 5 个事件埋点：`wallet_connected`、`referrer_bound`、`swap_submitted`、`swap_success`、`swap_failed`。  
- 若暂无法接入完整分析平台，可先用前端日志 + 链上 tx 导出做周报。
